# Yap Core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement Yap v1 per `yap-brief.md` — an API-first, self-hosted context system serving a REST API and an MCP surface from one process, onbox by default (SQLite + local disk), portable to Postgres + S3.

**Architecture:** One Node process built on fastmcp: MCP tools via `addTool`, REST via `addRoute`, both thin transports over a single core domain library. Storage behind two abstractions: structured data via Drizzle (sqlite + pg schema twins, shared query subset), blobs via FlyDrive (fs + s3) with app-served HMAC-signed links for local disk. Items are text-valued EAV; permissions are allow/deny grant rows resolved most-specific-wins.

**Tech Stack:** TypeScript (Node 22, ESM), fastmcp 4.x, zod, drizzle-orm 0.45 + drizzle-kit, better-sqlite3, pg, flydrive 2.x, Vitest 4.

## Key design decisions locked here

- **Capability names** (open-ended; these are the seeded known set): content `read_items, edit_items, edit_docs, read_files, edit_files, fire_hooks, edit_hooks`; container `create_bundles, edit_bundles, manage_roles, manage_space`.
- **Ownership confers nothing except personal space** (implicit all). To make created spaces usable, `space_create` seeds explicit allow rows for the creator for the full known set — auditable rows, not implicit logic.
- **"Bundle read access"** (gates `load_bundle`, `read_docs`, listing in `load_space`): user holds ≥1 effective allow of any capability on the bundle.
- **Grants:** `(user, resourceType, resourceId, capability, effect)` rows; POST accepts one capability or an array (creates N rows); DELETE by row id. "Role" is presented as the effective capability list.
- **EAV typing:** values TEXT; datatypes `text|number|boolean|date`; number compare via `CAST(value AS NUMERIC)` (works on both dialects); date stored ISO-8601 (lexicographic compare correct); boolean stored `true|false`. `neq`/comparisons only match items that *have* a value row.
- **Cursors:** opaque base64 `{o: offset}` — uniform across all list endpoints.
- **File storage keys:** `{spaceId}/{bundleId}/{fileId}`. Local upload `PUT /v1/files/:id/upload?token=`, local download `GET /v1/files/:id/download?token=` — both public routes gated by HMAC tokens (exp + single-use upload via reserved-status check). S3: FlyDrive presigned URLs.
- **Hook transport:** `{url, method, headers, bodyTemplate?}` JSON, AES-256-GCM-encrypted into one column; `{{param}}` substitution into url/headers/bodyTemplate; SSRF check at create AND fire (resolve DNS, deny private/link-local/loopback ranges unless host in config allowlist); 30s AbortController timeout; returns raw `{status, body}`.
- **Widget pointers:** per-call results carry `_meta: { "ui://...": params }`-style pointer object `{widget: "ui://yap/...", data}`; `call` tool result embeds per-call `_meta`. Origin pages at `/w/:widget?token=` with HMAC tokens embedding params.
- **MCP auth:** fastmcp `authenticate` reads `Authorization: Bearer` or `?key=`; sysadmin key is rejected on MCP. REST routes are `public: true` and do their own bearer auth (sysadmin where marked).

### Data Model (all milestones reference this)

```
users        id pk, name, created_at
access_keys  id pk, user_id fk→users(cascade), name, key_hash unique, created_at, revoked_at?
spaces       id pk, owner_id fk→users, name, description, keywords, context, personal int, created_at, updated_at
grants       id pk, user_id fk→users(cascade), resource_type 'space'|'bundle', resource_id,
             capability, effect 'allow'|'deny', created_at   [idx (user_id, resource_id, capability)]
bundles      id pk, space_id fk→spaces(cascade), name, description, docs, created_at, updated_at
item_types   id pk, bundle_id fk→bundles(cascade), name, created_at
properties   id pk, item_type_id fk→item_types(cascade), name, datatype, required int, sort_order
items        id pk, bundle_id fk(cascade), item_type_id fk(cascade), created_at, updated_at
item_values  id pk, item_id fk→items(cascade), property_id fk→properties(cascade), value
             [idx (property_id, value); idx (item_id)]
files        id pk, bundle_id fk(cascade), space_id, owner_id, status 'reserved'|'finalized',
             name, mime_type, size, storage_key, upload_consumed int, created_at, finalized_at?
hooks        id pk, bundle_id fk(cascade), name, description, params (json text),
             transport_encrypted, created_at, updated_at
user_docs    id pk, user_id fk(cascade), name, content, autoload int, created_at, updated_at
```

All ids are `crypto.randomUUID()`. Timestamps ISO-8601 text (portable across dialects).

### Task M1: Identity & permissions

**Files:** src/core/{errors,users,keys,spaces,grants,capabilities}.ts, src/rest/{auth,routes}.ts, tests/unit/capabilities.test.ts, tests/integration/{identity,spaces,grants}.test.ts.

- [ ] `core/users.createUser({name})` — creates user + **personal space** (`personal: 1`, name "Personal", owner) in one transaction; returns user + initial access key (secret shown once). `listUsers`, `getUser`, `deleteUser` (cascades; personal space deleted with user — owner deletion is the one path that removes it).
- [ ] `core/keys`: `createKey(userId, name)`, `listKeys`, `rotateKey(keyId)` (revoke old row by setting revoked_at, insert new, return new secret — immediate, no grace), `deleteKey`, `authenticate(bearer)` → userId via key_hash lookup where revoked_at null.
- [ ] `core/capabilities.resolveCapability(deps, userId, capability, {bundleId?, spaceId})`:
      1. personal space owned by user → allow (rows not consulted);
      2. bundle rows for (user,bundle,cap): any deny → deny; else any allow → allow;
      3. else space rows likewise; 4. else deny.
      `effectiveCapabilities(userId, space, bundle?)` → map for role display. **Deep unit coverage**: every state combo (allow/deny/absent × space/bundle), deny-beats-allow same level, bundle-deny-overrides-space-allow, bundle-allow-overrides-space-absent, personal-space implicit, default deny, unknown capability strings still resolve.
- [ ] `core/spaces`: create (seeds owner allow rows for KNOWN_CAPABILITIES), list-for-user (owner or any allow row at space/bundle level), get, update (reject personal rename; allow description/keywords/context edits? — reject *rename* and *delete* only, per brief), delete (reject personal), `manage_space` gate for update/delete by non-owner-row holders.
- [ ] `core/grants`: createGrant(spaceOrBundle, {userId, capability|capabilities, effect}) requiring `manage_roles` on the target (space grant→cap on space; bundle grant→cap on that bundle or its space); reject grants on personal spaces entirely. deleteGrant by id (same gate).
- [ ] REST: users (sysadmin only — constant-time compare; sysadmin rejected on MCP), keys (self), spaces, grants per the normative table; standard error shape `{error:{code,message,details?}}`; cursor pagination on lists.
- [ ] Integration tests (both adapters): provision user → personal space exists & rejects rename/delete/grants; key rotate revokes old immediately; sysadmin key fails on MCP authenticate; grant flows; acceptance-3-shaped check (space allow + bundle deny → blocked on that bundle, allowed on sibling, deciding row identifiable via a `decidedBy` field returned by a debug resolution helper).
- [ ] Commit per feature chunk.
